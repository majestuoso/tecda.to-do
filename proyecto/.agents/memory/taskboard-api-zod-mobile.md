---
name: TaskBoard api-zod mobile auth schemas
description: Mobile auth schemas aren't generated by Orval; api-server has no direct zod dependency.
---

The Orval codegen only generates schemas for endpoints that return JSON bodies. Mobile auth redirect/token endpoints weren't included in the generated output because the spec paths have 302 redirect responses (not JSON schemas).

**Why:** Orval skips generating Zod schemas for endpoints with non-JSON responses (like 302 redirects).

**How to apply:** For simple validation in api-server routes, use inline manual validation (check required fields manually) rather than importing zod directly — `@workspace/api-server` does NOT have `zod` as a direct dependency. If you need Zod in api-server, add `"zod": "catalog:"` to its `dependencies`.
